1. Executive Summary
Brookwell is a premium consulting firm specializing in AI-powered solutions for financial services. This PRD defines the design system, component architecture, and interaction patterns for a landing page that embodies sophistication through restraint—leveraging generous whitespace, monochromatic palettes, subtle textures, and purposeful animation to communicate expertise and trust.

The design philosophy draws inspiration from Attio's masterful use of negative space and Fyxer's refined minimalism, creating an experience that feels both cutting-edge and reassuringly professional.

2. Design Philosophy & Principles
2.1 Core Aesthetic Tenets
Restraint as Luxury: Every element must earn its place. Color is used sparingly—primarily through subtle gradients and the occasional accent. The absence of visual noise signals confidence and expertise.

Purposeful Motion: Animations serve function, not decoration. Elements fade in as users scroll, creating a sense of discovery. Lottie animations within bento cards provide visual interest without overwhelming.

Tactile Depth: Subtle noise textures, soft shadows, and layered surfaces create a sense of materiality. The page should feel like high-quality paper stock—substantial but not heavy.

Spatial Harmony: Generous margins and padding create breathing room. Content blocks are separated by significant vertical rhythm (120-200px between major sections). Grid systems provide invisible structure.

8. Animation Specifications
8.1 Scroll-Triggered Animations
Fade-In Up:


@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
Duration: 600ms
Easing: cubic-bezier(0.25, 0.46, 0.45, 0.94)
Trigger: When element is 20% visible
Stagger Pattern:

Base delay: 0ms
Increment: 100ms per element
Max delay: 400ms (to avoid long waits)
8.2 Lottie Animation Guidelines
Placement:

Hero: Optional ambient background animation
Bento cards: Centered, 200-300px dimensions
Process steps: 80-100px icon size
Behavior:

Autoplay when in viewport
Loop for ambient/background
Play-once for process steps (replay on scroll back)
Pause when tab is not active (performance)
File Requirements:

Max file size: 100KB per animation
Frame rate: 30fps
Optimize with bodymovin settings

Key Responsive Behaviors:

< 1024px: Hamburger menu
= 1024px: Full horizontal nav

Hero:

< 768px: Stack all content, reduce headline size
= 768px: Standard layout

Bento Grids:

< 768px: Single column
768-1024px: 2 columns
= 1024px: Full grid

Typography Scaling:

Headline: 72px → 48px → 36px (xl → md → sm)
Body: Maintains 16px base, line-height increases on mobile
10. Performance Requirements
Target Metrics:
First Contentful Paint: < 1.5s
Largest Contentful Paint: < 2.5s
Cumulative Layout Shift: < 0.1
Time to Interactive: < 3.5s
Optimization Strategies:
Lazy load images and Lottie files below fold
Use Intersection Observer for scroll animations
Preload critical fonts (Inter 400, 500, 600)
Optimize Lottie files (remove unused layers)
Use CSS transforms for animations (GPU acceleration)
Implement proper image srcset for responsive images
11. Accessibility Requirements
Color contrast: Minimum 4.5:1 for body text
Focus states: Visible outline on all interactive elements
Animation: Respect prefers-reduced-motion
Semantic HTML: Proper heading hierarchy
ARIA labels: On all interactive components
Keyboard navigation: Full support
Screen reader: All images have descriptive alt text
